HorusEye: A Realtime IoT Malicious Traffic Detection Framework using Programmable Switches

是一个两阶段的异常检测框架，用于检测物联网恶意流量，异常检测
第一阶段，在数据层面实现初步的突发级异常检测
第二阶段在控制层面，使用 flow-level 特征和一个深度检测模型对报告的可疑流量进行更彻底的异常检测

Introduction

无监督学习具有识别未见攻击的能力
可编程交换机带来了更高的吞吐量，更低的延迟和更快的数据包转发速率
实现该系统的三个挑战

  • 在仅支持简单指令且资源有限的可编程交换机上部署具有高异常召回率和卸载能力的无监督模型是困难的
  • 在有限内存的交换机上提取和维持所需的流量特征是具有挑战性的
  • 由于控制平面是主要的吞吐量瓶颈，因此使用高吞吐量深度模型实现低假阳性率是具有挑战性的

本文的工作：

  • 将具有许多隔离树(iTrees)的隔离森林(iForest)转换为一组小的白名单规则，可以很容易的部署在可编程交换机上。
  • 设计了一种可部署在可编程交换机上的物联网流量特征提取方案，提出了bi-hash和double hash table两种方法来匹配双向流，计算复杂度为$O(1)$。
  • 设计了一个非对称轻量级自编码器，并在控制平面上实现了模型量化，从而在不显著降低检测性能的情况下实现了每秒数百万数据包的吞吐量。

    整体结构

Overview of HorusEye

包括两个部分

  • Gulliver Tunnel, data plane, programmable switches
  • Magnifier, control plane, x86 server

Gulliver Tunnel detection
首先提取 burst-level 特征，基于 iForest 模型检测异常，但是因为很大，在单个交换机上部署不实际，所以将这个训练好的模型使用算法转换为白名单规则

Magnifier detection
控制平面的Magnifier接收到来自数据平面的可疑流ID后，首先将可疑流ID与其可疑流表中的流ID进行匹配。如果流量的可疑异常频率超过阈值，则Magnifier将使用轻量级深度学习模型从历史镜像流量中进一步分析流量模式。最后，将确认的恶意流ID添加到数据平面的黑名单中，以阻断或限制该恶意流的速率。此外，当获得新的流信息时，我们在控制平面上更新了无监督深度模型和ifforest模型。

IForest Rule Generation

我们观察到，每个iTree本质上都进行了一轮超维特征空间划分。也就是说，每个iTree通过二叉树分支将超维特征空间划分为超立方体(称为iTree超立方体)，如图3(a)和3(b)所示。为了在数据平面上部署ifforest规则，我们聚合了ifforest中所有itree的空间划分，即分支，并在超维特征空间中获得更精细的超立方体(称为ifforest超立方体)，如图3(c)所示。具体来说，观察到聚合后每个超立方体的标签一致性属性如下。

  • iForest超立方体内的所有样本都具有相同的异常得分。
  • 由于决策面是实数，而特征多是整数，所以将决策面向下取整

在得到具有整数边界的超立方体后，我们需要确定每个超立方体中样本的标签。根据标签一致性的性质，可以使用超立方体中的任何数据点来获得超立方体的标签。在不损失一般性的情况下，对于每个超立方体，我们选择每个特征维中值最大的顶点，即右界。然后，我们使用ifforest模型预测的顶点标签作为超立方体的标签。最后，标有“normal”标签的超立方体将被转换为数据平面上的白名单规则

Burst Feature Extractor

突发特征，就是从突发流量提取的特征
突发流量就是在短时间内连续的大量数据包
使用突发流量判断事件活动需要大量资源来维持长序列包
为了解决这个问题，我们提出了突发分割来减少keepalive流量对长期资源的占用，双哈希来实现双向流匹配，双哈希表来解决哈希冲突问题。它将在以下小节中进行描述。
Bi-hash
传统的双向流匹配是计算五元组的哈希函数，然后通过字典来维护反向流的ID
这里使用双哈希异或来维护
Double hash table
分配两个位数相等的哈希表，如果第一个表命中，则使用新的哈希函数计算第一个哈希值的哈希值，在第二个表中查，碰撞率小10倍

Hardware Implementation


看不太懂
分为三个step

  • Parsing：按照包的突发特征觉得是否进行异常检测或者更新突发特征信息，交换机以标志的形式将决策添加到额外的元数据中，并将数据包重新提交给pipeline
  • Detection and update：第二次收到数据包时，交换机执行异常检测或者特征更新
  • Post-processing：对于镜像包，如果流正常，则丢弃有效负载并将报头上传到服务器以进行存储。否则，我们封装最后一个镜像的包头，将异常的突发信息(突发大小，数据包数量)添加到新的包头中，并将数据包发送到控制平面。然后，控制平面使用交换机上报的异常镜像报文的五元组提取所有存储的历史报文，然后进行进一步的检测。

Robust Multi-tab Website Fingerprinting Attacks in the Wild

现在的WF攻击在多标签浏览会话中准确识别网站方面存在严重限制，其中不再保留单个网站的整体模式，并且客户端打开的标签数量是先验未知的。
WF攻击背后的基本原理是，每个网站的内容都有与其他网站不同的独特的流量模式

现有的WF攻击的有效性依赖于一个普遍但是不切实际的假设：客户端在一个浏览会话中只访问一个网页。
存在的问题

  • 需要开了几个标签页的先验知识
  • 对防御机制没有弹性
  • 对浏览器选项卡的数量增加，有效性进一步降低

本文的工作

  • 将多标签页WF攻击规约成一个多标签分类问题
  • 尽管网站的干净和整体流量模式不再在多选项卡浏览会话中保留，但仍然有可能从多个短流量段中提取网站的多个局部模式。

  • 一个系统
  • 一个数据集
